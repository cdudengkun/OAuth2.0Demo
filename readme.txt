1、本demo实现的是OAuth2.0的授权码模式（authorization code）;
2、授权码模式的有三个角色：第三方客户端、用户、认证服务端;
3、授权码的流程如下：
    A、第三方应用想要接入认证服务器，需要去认证服务器提供的应用注册网站，注册自己，并将生成的client_id保存到自己系统;
    B、第三方客户端需要获取用户信息，引导用户前往任务服务器端的页面;
    C、认证服务器尝试获得用户授权，如果用户在认证服务器未登录那么需要先引导用户登录，之后再进行授权;
    D、认证服务器授权成功之后生成授权码返回到C步骤中第三方应用传递过来的回调地址;
    E、第三方应用使用授权码在认证服务器使用授权码去换取授权令牌;
    F、第三方应用带上授权另外去获取用户信息;
